# Design System

> 设计语言参考：Apple iOS 原生应用设计范式
> 核心原则：极简、列表式、无卡片、高信息密度、一致性

## 1. 色彩系统

### 1.1 全局色板

```
#007AFF    品牌色/强调色 (iOS Blue) — 交互焦点、高亮、选中态
#0056CC    品牌色深色 — hover/active 状态
#FF3B30    收藏色 (iOS Red) — 收藏/喜欢状态
#1D1D1F    文字主色 — 标题、重要文本
#3A3A3C    文字次色 — 正文、例句英文
#6E6E73    文字辅助色 — 描述、次要信息、未选中按钮
#C7C7CC    文字弱色 — 分隔符、未选中图标
#FAFAFA    页面背景 — 所有页面统一底色
#FFFFFF    卡片/模块背景 — 需要区分的区域
#F5F5F7    交互反馈背景 — hover/active 态、评论预览底块
#E5E5EA    分隔线色 — 列表项分隔
#E8F0FE    图标背景（蓝色） — 句型学习模块
#F3E8FF    图标背景（紫色） — 学习交流模块
#E6F9F0    图标背景（绿色） — 文章分享模块
```

## 2. 排版系统

### 2.1 字号与字重

| 层级            | 字号   | 字重             | 颜色        | 用途           |
| ------------- | ---- | -------------- | --------- | ------------ |
| H1 页面标题       | 28px | bold (700)     | `#1D1D1F` | 页面级标题、登录页大标题 |
| H2 区块标题       | 18px | semibold (600) | `#1D1D1F` | 句型标题、列表项标题   |
| H3 导航标题       | 17px | semibold (600) | `#1D1D1F` | 页面头部标题       |
| Body 正文       | 16px | normal (400)   | `#3A3A3C` | 例句英文         |
| Body Small    | 14px | normal (400)   | `#6E6E73` | 例句中文、描述、句型翻译 |
| Caption       | 13px | medium (500)   | `#6E6E73` | 评论预览、按钮小字    |
| Caption Small | 12px | medium (500)   | `#6E6E73` | 标签、徽章        |

### 2.2 行高规则

- 英文正文（例句）：`leading-snug` (1.25)
- 中文正文（翻译）：`leading-snug` (1.25)
- 标题：`leading-tight`
- 紧凑列表项：`leading-none` (图标/序号)

### 2.3 字间距

- 标题：`tracking-tight`
- 正文：默认 (无额外设置)

### 2.4 命名区分规范

句型标题 vs 例句的视觉区分（核心）：

- 句型标题：`18px font-semibold #1D1D1F`
- 例句英文：`16px normal #3A3A3C`
- 例句中文：`14px normal #6E6E73`

## 3. 间距系统

### 3.1 基础间距单位

使用 Tailwind CSS v4 默认间距系统，基础单位为 4px。

### 3.2 列表间距

| 场景        | 间距          | 说明            |
| --------- | ----------- | ------------- |
| 列表项上下内边距  | `py-4`      | 句型之间垂直间距      |
| 例句间距      | `space-y-1` | 例句之间的紧凑间距     |
| 例句与标题间距   | `mt-2`      | 标题到例句组        |
| 例句与操作按钮间距 | `mt-2`      | 例句组到收藏/评论     |
| 操作按钮与评论间距 | `mt-3`      | 操作到评论预览       |
| 例句缩进      | `ml-[44px]` | 对齐 emoji 图标右侧 |

### 3.3 模块间距

| 场景      | 间距          | 说明        |
| ------- | ----------- | --------- |
| 模块列表项   | `space-y-0` | 首页模块之间无间距 |
| 首页模块内间距 | `px-5 py-5` | 首页内边距     |

## 4. 圆角系统

| 圆角  | 值                | 用途           |
| --- | ---------------- | ------------ |
| 小圆角 | `rounded-[8px]`  | 图标背景、小标签     |
| 中圆角 | `rounded-[12px]` | 按钮、输入框、卡片    |
| 大圆角 | `rounded-[16px]` | 大模块、对话框      |
| 胶囊  | `rounded-full`   | 小按钮、标签、数字导航项 |

## 5. 阴影系统

> 极简风格，阴影使用极少。

- 默认不添加阴影，通过色彩和分隔线实现层级区分
- 如有悬浮/弹出层需求，使用最轻量的阴影

## 6. 布局模式

### 6.1 列表 + 分隔线（核心布局模式）

**不使用卡片容器**。所有内容通过列表式布局呈现，用分隔线和间距区分层级。

```
标题 18px semibold #1D1D1F
描述 14px #6E6E73
───────────────────── ← #E5E5EA 分隔线
标题
描述
─────────────────────
```

### 6.2 模块入口（首页）

图标背景 + 文字 + 箭头，无卡片容器：

```
┌──┐
││  标题          →
└──┘  描述
─────────────────────
```

- 图标背景：40x40px，圆角 12px
- 图标颜色：根据模块使用不同色
- 箭头：`#C7C7CC`，16px

### 6.3 句型列表（核心内容页）

```
[emoji] 句型标题           [折叠箭头]
        句型翻译
    ───────────────────── ← 分隔线

        例句 1 英文
        例句 1 中文
        例句 2 英文
        例句 2 中文
        ...

        ♥ 收藏 · ○ 评论 (2)

        [浅灰底块]          ← 评论预览
        用户名: 评论内容
```

### 6.4 评论预览底块

- 背景：`#F5F5F7`
- 圆角：`rounded-[12px]`
- 内边距：`px-3 py-2.5`
- 用户名：`13px #1D1D1F font-medium`
- 内容：`13px #6E6E73`

## 12. 设计约束

### 12.1 禁止使用

- ❌ 卡片容器（白色圆角底块包内容）
- ❌ emoji 图标（首页模块入口除外）
- ❌ 渐变背景
- ❌ 复杂阴影
- ❌ 装饰性元素（波纹、曲线、波浪等）
- ❌ 圆点指示器（TabBar 不显示选中指示点）
- ❌ 缩放动画（TabBar 不使用 scale）

### 12.2 必须遵守

- ✅ 使用列表 + 分隔线布局
- ✅ 通过字号、字重、颜色区分层级
- ✅ 保持页面背景统一为 `#FAFAFA`
- ✅ 图标统一使用 `lucide-react`
- ✅ 按钮交互反馈使用 `opacity-50` 或背景色变化
- ✅ 高信息密度，紧凑间距

## 13. 设计原则

### 13.1 组件设计原则

**1. 功能复用优先**
相同功能的 UI 应使用同一实现，通过配置参数区分差异，而非创建新组件。

**2. 抽象层级清晰**
- 容器组件：负责布局和结构（如详情页、抽屉）
- 展示组件：负责内容渲染（如列表项、评论项）
- 基础组件：原子级 UI 元素（如按钮、输入框）

**3. 样式与数据分离**
组件样式应独立于业务数据，通过 props 传入数据，保持样式内聚。

### 13.2 新增功能设计流程

**Step 1: 模式匹配**
新功能应先匹配现有设计模式：
- 列表展示 → 参照第 12.1 节主内容列表
- 详情展示 → 参照第 11.1 节全屏详情容器
- 评论功能 → 参照第 12.2/12.3 节评论模式
- 快捷操作 → 参照第 11.3 节悬浮操作按钮

**Step 2: 样式检查**
- 颜色是否来自第 1 节色板？
- 字号是否遵循第 2 节规范？
- 间距是否使用第 3 节系统？
- 按钮是否符合第 7 节定义？

**Step 3: 一致性验证**
- 是否与现有同类型功能视觉一致？
- 交互反馈是否统一（`opacity-50`）？
- 是否避免了第 12.1 节的禁止项？

### 13.3 设计决策优先级

1. **一致性 > 创新性** - 优先复用现有模式
2. **简洁性 > 完整性** - 保持极简，避免过度设计
3. **可用性 > 美观性** - 功能优先，视觉服务于功能
